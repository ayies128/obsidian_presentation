# Obsidian vs Notion

## なぜObsidianを選ぶのか？

---

## 比較表

| 項目 | Obsidian | Notion |
|------|----------|--------|
| データ保存 | **ローカル（自分のPC）** | クラウド（Notion社サーバー） |
| ファイル形式 | **Markdown（.md）** | 独自形式 |
| オフライン | **完全対応** | 制限あり |
| 速度 | **高速** | ネットワーク依存 |
| AI連携 | **Claude Code対応** | Notion AI（別料金） |
| 料金 | **無料**（Sync有料） | 無料〜有料 |

---

## Obsidian のメリット

### 1. ローカルファイル = Claude Code と相性抜群

```
プロジェクト/
├── src/
├── docs/          ← Obsidian Vault
│   └── specs.md   ← Claude Code で直接編集可能
└── package.json
```

> Claude Codeがファイルを直接読み書きできる！

---

### 2. Markdownだから汎用性が高い

- **GitHub** でそのまま表示される
- **VSCode** でも編集できる
- **他ツールへの移行**が簡単
- **Git管理**が自然にできる

---

### 3. オフラインで完全動作

- ネット接続不要で編集可能
- 飛行機の中でも作業できる
- サーバー障害の影響を受けない

---

### 4. 高速な動作

- ローカルファイルなので**読み込み爆速**
- 大量のノートがあっても快適
- 検索も一瞬

---

## Obsidian のデメリット

### 1. 初期設定が必要

- プラグインの導入が必要な機能もある
- 自分でカスタマイズする必要がある

### 2. 共有に一手間

- Obsidian Sync（有料）または GitHub が必要
- Notionのようにリンク共有で即公開は不可

### 3. リッチな埋め込みが弱い

- データベース機能はNotionが強い
- 埋め込みウィジェットは少ない

---

## Notion のメリット

### 1. すぐに共有できる

- リンク1つで誰でも閲覧可能
- 権限管理も簡単

### 2. データベース機能が強力

- テーブル、カンバン、カレンダー
- リレーション、ロールアップ

### 3. 設定不要で始められる

- アカウント作成 → すぐ使える
- 直感的なUI

---

## Notion のデメリット

### 1. Claude Code と連携しにくい

```
❌ Claude Code → Notion API → 複雑な設定が必要
✅ Claude Code → ローカル.md → 直接編集
```

### 2. データが手元にない

- エクスポートしないとバックアップできない
- サービス終了リスク

### 3. オフラインに弱い

- ネット必須（キャッシュはあるが制限あり）

### 4. 独自形式でロックイン

- 他ツールへの移行が面倒

---

## 結論：用途で使い分け

| 用途 | おすすめ |
|------|----------|
| **開発ドキュメント** | Obsidian（Claude Code連携） |
| **個人のナレッジ管理** | Obsidian |
| **チームのタスク管理** | Notion |
| **社外への共有** | Notion |
| **両方使う** | Obsidian（仕様）+ Notion（タスク） |

---
